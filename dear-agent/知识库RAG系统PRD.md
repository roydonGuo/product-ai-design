# 知识库（RAG）系统 — 产品需求文档

> **项目名称**：dear-agent 知识库系统（KnowFlow）  
> **版本**：v1.0  
> **面向用户**：C 端个人用户  
> **核心定位**：为 AI Agent 提供个人知识管理（RAG）能力，让 Agent 理解并使用用户的私有知识

---

## 1. 产品概述

### 1.1 产品背景

dear-agent 是一款面向 C 端用户的桌面端 AI 陪伴智能体应用。用户在与 AI Agent 的交互中，需要 Agent 能够理解和使用用户的私有知识（如学习笔记、工作文档、个人资料等）。知识库系统即为 Agent 提供「外挂大脑」—— 用户将文档导入知识库，Agent 通过 RAG（检索增强生成）技术检索相关内容来增强回答质量。

### 1.2 产品目标

- 提供一个直观、美观的个人知识管理平台
- 支持知识库的分级分类组织（知识库 → 文件夹 → 文件）
- 提供流畅的 Markdown 文档编辑体验
- 落地 RAG 核心链路：文档导入 → 向量化 → 检索 → 增强生成
- 与 dear-agent 主应用的 AI 对话能力深度集成

### 1.3 功能全景图

```
知识库系统 (KnowFlow)
├── 知识库列表页
│   ├── 分类标签栏（筛选）
│   ├── 搜索栏
│   ├── 创建知识库
│   ├── 卡片列表展示（网格模式）
│   └── 触底分页加载更多
├── 知识库详情页
│   ├── 左侧：树形文件/文件夹结构
│   │   ├── 文件夹展开/折叠
│   │   ├── 新建文件/文件夹
│   │   ├── 重命名
│   │   ├── 拖拽移动
│   │   └── 删除
│   ├── 右侧：Markdown 内容区
│   │   ├── 查看模式（渲染展示）
│   │   ├── 编辑模式（富文本编辑器）
│   │   └── 悬浮操作工具栏
│   └── 顶部导航栏（返回、知识库名称、操作）
└── RAG 引擎
    ├── 文档向量化（Embedding）
    ├── 向量数据库存储
    └── 语义检索（与 AI 对话集成）
```

## 2. 功能规格

### 2.1 知识库列表页

#### 2.1.1 分类管理

| 功能 | 描述 |
|------|------|
| 默认分类 | 系统提供一组默认分类：「全部」「学习」「工作」「生活」「项目」「其他」 |
| 自定义分类 | 用户可新增/编辑/删除自定义分类标签 |
| 分类筛选 | 点击分类标签，知识库列表按所属分类过滤；「全部」显示所有知识库 |
| 多分类 | 一个知识库可属于多个分类（多对多关系） |

#### 2.1.2 知识库创建

| 字段 | 类型 | 必填 | 说明 |
|------|------|------|------|
| 封面 | 选择器 | 否 | 从预设图案库中选择封面风格（抽象几何图案/渐变色），支持上传自定义封面 |
| 标题 | 文本 | 是 | 知识库名称，最多 50 字 |
| 所属分类 | 多选标签 | 否 | 从已有分类中选择，可多选 |
| 描述 | 多行文本 | 否 | 知识库简短介绍，最多 200 字，列表页最多显示 2 行 |

创建入口：列表页右上角的「+ 新建」按钮，弹出创建模态框。

#### 2.1.3 卡片列表展示

采用**网格模式（Grid）**展示，每个知识库为一个独立卡片：

```
┌─────────────────────────────────────┐
│         ┌──────┐                    │
│         │封面图 │                    │
│         └──────┘                    │
│  📁 标题                    [···]  │
│  描述文字最多显示两行...             │
│  🏷️ 分类1  分类2                    │
│  🕐 2026-05-13更新  📄 12个文件      │
└─────────────────────────────────────┘
```

卡片信息包含：
- **封面**：卡片顶部横幅展示，区分每个知识库的视觉标识
- **标题**：知识库名称
- **所属分类**：以标签（Tag Chip）形式展示在标题下方
- **描述**：摘要展示，超长截断（最多 2 行）
- **创建时间**：显示于卡片底部
- **更新时间**：显示于卡片底部
- **文件数量**：知识库下所有文件总数
- **操作菜单**：卡片右上角 `···` 按钮，下拉菜单包含「编辑信息」「删除」

#### 2.1.4 分页与加载

- 采用**触底加载更多**（Infinite Scroll）的分页方式
- 每页加载 20 个知识库
- 正在加载时，底部显示骨架屏或加载指示器
- 所有知识库加载完毕时，底部显示「已加载全部」
- 搜索/筛选时重置分页

#### 2.1.5 搜索

- 提供搜索输入框，按知识库标题和描述进行全文搜索
- 输入即搜（debounce 300ms），搜索结果以卡片列表形式展示
- 搜索时分类筛选仍生效（搜索 + 分类的组合筛选）

#### 2.1.6 视图切换

- 支持网格视图（卡片式，默认）和列表视图（紧凑型）切换
- 切换按钮位于搜索栏附近

### 2.2 知识库详情页

点击知识库卡片后，进入该知识库的详情页，采用**左右分栏布局**。

#### 2.2.1 顶部导航

```
[← 返回]  知识库名称  [操作按钮组]
```

- 返回按钮：回到知识库列表页
- 知识库名称：当前知识库标题
- 操作按钮组：编辑知识库信息、AI 向量化全部文件、删除知识库

#### 2.2.2 左侧：树形文件结构（Tree Panel）

类似 VSCode 的项目文件浏览器，展示该知识库下的文件夹和文件。

**视觉规范：**
- 宽度：300px，支持拖拽调整宽度（最小 220px）
- 每行缩进体现层级（每级 20px）
- 文件夹图标 📁（展开时 📂），文件图标 📄（Markdown 文件标记）
- 选中节点高亮背景

**交互行为：**

| 操作 | 行为 |
|------|------|
| 点击文件夹 | 展开/折叠子节点 |
| 点击文件 | 右侧显示该文件 Markdown 内容 |
| 右键菜单 | 呼出上下文菜单（重命名、删除、移动） |
| 拖拽 | 拖拽文件/文件夹到目标文件夹以移动 |
| 搜索 | 树顶搜索框，按文件名实时过滤 |

**节点操作按钮（每个节点悬停显示）：**
- 文件夹：`[➕ 新建子文件夹]` `[📝 重命名]` `[🗑️ 删除]`
- 文件：`[📝 重命名]` `[🗑️ 删除]`

**创建操作（树顶工具栏）：**
- `[📁 新建文件夹]` 按钮：在当前选中的文件夹下创建子文件夹，或在根层级创建
- `[📄 新建文件]` 按钮：在当前选中的文件夹下创建 Markdown 文件，或在根层级创建
- 新建时直接进入「重命名」状态（内联编辑模式）

#### 2.2.3 右侧：Markdown 内容区

默认显示当前选中文件的 Markdown 渲染内容，支持两种模式：

**查看模式（默认）：**
- 渲染 Markdown 为格式化 HTML 展示
- 支持标准 Markdown 语法：标题、列表、表格、代码块、引用、图片、链接等
- 代码块支持语法高亮
- 选中文本时弹出浮动格式工具栏（加粗、斜体、链接等）

**编辑模式：**
- 采用主流 Markdown 编辑器（如 Toast UI Editor / Milkdown / CodeMirror）
- 支持所见即所得（WYSIWYG）编辑
- 支持 Split View：左侧源码，右侧预览
- 自动保存（debounce 2s，保存至本地/远程）
- 快捷键支持：`Ctrl+S` 保存、`Ctrl+B` 加粗、`Ctrl+I` 斜体等
- 支持插入图片（粘贴或拖拽上传）

          ├─────┤
          │ 🤖  │  AI对话 — 与该文件内容进行对话
          ├─────┤

每个按钮的详细功能：

#### ① 编辑按钮（Edit）

| 项目 | 说明 |

| 交互 | 点击某标题，滚动到文档对应位置并高亮 |
| 定位 | 大纲面板位于内容区右侧，宽度 240px |

#### ③ 分享按钮（Share）

| 项目 | 说明 |
|------|------|
| 触发行为 | 点击弹出分享对话框 |
| 分享方式 | ① 复制 Markdown 原文链接（需后端支持） ② 复制分享链接（只读访问） ③ 导出为图片分享 |
| 权限 | C 端个人场景，分享为只读访问，不可编辑 |
| UI | 分享弹窗包含链接预览、权限说明、复制按钮 |

#### ④ 导出按钮（Export）

| 项目 | 说明 |
|------|------|
| 触发行为 | 点击展开导出格式菜单 |
| 支持格式 | Markdown (.md)、PDF、HTML、纯文本 (.txt) |
| 导出行为 | 浏览器直接下载文件 |
| PDF 导出 | 使用 HTML → PDF 转换（如 html2pdf.js），保留 Markdown 渲染样式 |

#### ⑤ 喜欢按钮（Like）

| 项目 | 说明 |
|------|------|
| 触发行为 | 点击切换喜欢/取消喜欢状态 |
| 状态 | ❤️ 实心 = 已喜欢，🤍 空心 = 未喜欢 |
| 功能 | 收藏的文档可在独立「我的收藏」列表中查看 |
| 反馈 | 点击时给予微交互动效（如心形填充动画） |

#### ⑥ AI 对话按钮（AI Chat）

| 项目 | 说明 |
|------|------|
| 触发行为 | 点击在右侧或底部打开 AI 对话面板（参考主应用的对话界面） |
| 上下文 | 自动以当前文件全文作为对话的上下文 |
| 使用场景 | 用户可与 Agent 就当前文档内容进行讨论：总结、提问、扩写、翻译等 |
| 对话面板 | 浮动面板，可拖拽位置，可调整大小，可最小化 |
| 与主应用关系 | 这是主应用 AI 对话的上下文扩展，而非替代 |

#### ⑦ AI 向量化按钮（RAG / Vectorize）

| 项目 | 说明 |
|------|------|
| 触发行为 | 点击将该文件纳入 RAG 向量索引 |
| 状态指示 | 灰色 = 未向量化；蓝色 = 已向量化；旋转 = 正在向量化 |
| 向量化流程 | 读取 Markdown 全文 → 文本分块（Chunking）→ Embedding 向量化 → 存入向量数据库 |
| 取消向量化 | 再次点击可取消该文件的向量化（从索引中移除） |
| 批量操作 | 知识库详情页顶部支持「全部向量化」操作 |
| 文件变更同步 | 已向量化的文件被编辑后，自动重新向量化（增量更新） |

### 2.4 RAG 引擎规格

#### 2.4.1 核心流程

```
用户提问
    │
    ▼
┌─────────────────┐
│   Query 理解     │  ← 对用户问题进行改写/扩展
└────────┬────────┘
         ▼
┌─────────────────┐
│  语义检索        │  ← 在向量数据库中检索最相关文档片段
└────────┬────────┘
         ▼
┌─────────────────┐
│  结果重排序      │  ← 对检索结果按相关性排序
└────────┬────────┘
         ▼
┌─────────────────┐
│  LLM 增强生成    │  ← 将检索结果作为上下文注入 Prompt
└────────┬────────┘
         ▼
      回答用户
```

#### 2.4.2 文档处理

| 步骤 | 说明 |
|------|------|
| 文本提取 | 解析 Markdown 文件为纯文本 |
| 文本分块（Chunking） | 按段落/标题分割，每块约 500-1000 tokens，块间重叠 10% |
| Embedding | 使用 Embedding 模型将文本块转为向量 |
| 向量存储 | 存入向量数据库（Chroma / FAISS 等本地方案） |
| 索引管理 | 记录每个向量对应的文件 ID、块序号、知识库 ID |

#### 2.4.3 检索策略

- 默认检索每个知识库 Top-K 结果（K=5）
- 支持跨知识库检索（搜索全部已向量化文档）
- 检索结果附加元数据（文件名、路径、知识库名）供 LLM 引用来源

## 3. 数据模型

### 3.1 知识库（KnowledgeBase）

```typescript
interface KnowledgeBase {
  id: string;                // 唯一标识
  title: string;             // 标题
  description: string;       // 描述
  cover: string;             // 封面图案 ID
  categoryIds: string[];     // 所属分类 ID 列表
  createdAt: string;         // ISO 时间戳
  updatedAt: string;         // ISO 时间戳
  fileCount: number;         // 文件数量（冗余，便于展示）
}
```

### 3.2 分类（Category）

```typescript
interface Category {
  id: string;                // 唯一标识
  name: string;              // 分类名称
  color: string;             // 标签颜色
  createdAt: string;
}
```

### 3.3 文件夹（Folder）

```typescript
interface Folder {
  id: string;
  kbId: string;              // 所属知识库 ID
  parentId: string | null;   // 父文件夹 ID（null = 根层级）
  name: string;              // 文件夹名称
  createdAt: string;
  updatedAt: string;
}
```

### 3.4 文件（Document）

```typescript
interface Document {
  id: string;
  kbId: string;              // 所属知识库 ID
  folderId: string | null;   // 所属文件夹 ID（null = 根层级）
  title: string;             // 文件名（不含 .md 后缀）
  content: string;           // Markdown 内容
  isFavorite: boolean;       // 是否喜欢/收藏
  isVectorized: boolean;     // 是否已向量化
  vectorizedAt: string | null; // 向量化时间
  createdAt: string;
  updatedAt: string;
}
```

### 3.5 数据关系

```
Category ──╼ KnowledgeBase ──╼ Folder ──╼ Document
                 │                            │
                 │                            ├── VectorIndex (RAG)
                 └──╼ Document (根层级文件)     │
                                              └── Favorite (收藏)
```

## 4. 交互流程

### 4.1 用户查看知识库列表

```
启动知识库
  → 进入知识库列表页
  → 默认展示「全部」分类下的所有知识库卡片
  → 用户可点击分类标签筛选
  → 向下滚动触底加载更多
  → 点击搜索框输入关键词搜索
  → 点击某知识库卡片 → 进入详情页
```

### 4.2 用户创建知识库

```
点击「+ 新建」按钮
  → 弹出创建模态框
  → 选择封面图案
  → 输入标题
  → 选择分类（可选）
  → 输入描述（可选）
  → 点击「创建」→ 新知识库出现在列表首位
  → 点击新知识库 → 进入空知识库详情页
```

### 4.3 用户在知识库中管理文件

```
进入知识库详情页
  → 左侧树形结构显示根节点
  → 点击「新建文件夹」→ 输入名称 → 新增节点
  → 点击「新建文件」→ 输入名称 → 新增 Markdown 文件节点
  → 点击文件夹 → 展开子节点
  → 拖拽文件到文件夹 → 移动
  → 右键 → 重命名/删除
  → 点击文件 → 右侧展示 Markdown 内容
```

### 4.4 用户编辑和操作 Markdown 文件

```
点击某 Markdown 文件
  → 右侧渲染展示文档内容（查看模式）
  → 点击悬浮工具栏「编辑」按钮
  → 切换到编辑模式（WYSIWYG）
  → 编辑完成后 Ctrl+S 保存
  → 点击「完成」退出编辑模式
  → 勾选「大纲」查看文档结构
  → 点击「AI 对话」打开对话面板，基于文档内容提问
  → 点击「向量化」将该文档加入 RAG 索引
```

## 5. UI 设计规范

### 5.1 设计语言

- **风格**：优雅极简主义、玻璃拟态（Glassmorphism）、柔和渐变
- **强调色**：蓝色系（主色调 #2563EB）
- **字体**：系统字体栈（SF Pro / -apple-system）
- **圆角**：精心打磨的圆角（卡片 12px，按钮 8px，标签 100px）
- **阴影**：轻盈通透的阴影层次

### 5.2 布局规格

| 区域 | 规格 |
|------|------|
| 列表页 | 顶部导航栏 52px + 工具栏 + 标签栏 + 卡片网格（自适应列数） |
| 详情页 | 左树 300px + 右内容区（自适应），顶部导航栏 52px |
| 模态框 | 居中弹窗，宽 420px，最大宽 90vw |
| 悬浮工具栏 | 右侧固定定位，按钮间距 8px，圆形图标按钮 36px |

### 5.3 暗色模式

- 支持深色/浅色主题切换
- 与 dear-agent 主应用主题联动
- 使用 OKLCH 色彩空间确保深浅主题下的视觉一致性

### 5.4 空状态

- 无知识库时：展示引导性空状态插图 + 提示文字 + 「创建第一个知识库」按钮
- 无文件时：在树形区域显示「暂无文件，点击新建」引导
- 搜索无结果时：展示「未找到匹配的知识库」

## 6. 非功能需求

### 6.1 性能

- 知识库列表首次加载 < 1s（本地数据）
- Markdown 内容渲染 < 500ms（10KB 以内文档）
- 树形结构展开/折叠无感知延迟
- 触底加载流畅，无滚动卡顿

### 6.2 离线能力

- 知识库元数据和文档内容本地持久化（IndexedDB）
- 离线可查看、编辑全部知识库内容
- 在线时同步至云端（未来扩展）

### 6.3 兼容性

- 桌面端 Chrome / Edge / Safari 最新版本
- 分辨率适配 1280×720 及以上

### 6.4 安全性

- 所有用户数据仅本地存储（当前版本）
- Markdown 内容渲染时防止 XSS 注入

## 7. 与主应用的集成关系

```
dear-agent 主应用
├── 首页（AI 对话）
├── 技能管理
├── 知识库管理 ← 本 PRD
│   ├── RAG 检索接口 → 供 AI 对话调用
│   └── 文件引用 → 对话中可引用知识库内容
└── 创作页
```

- 知识库详情页的「AI 对话」按钮复用了主应用的对话能力，但上下文限定为当前文档
- 主应用对话时，Agent 可自动检索知识库中已向量化的内容作为参考
- 对话中引用知识库内容时，标注来源（文件名 + 路径）
